# Example: what your files look like after onboarding

Everything in this folder is **fictional**. "Priya" does not exist, "EXAMPLECO" is not a real stock, and none of the numbers are advice. The point is to show the target state: what the agent should have produced in `memory/` after onboarding and a few weeks of normal use.

Read it in this order:

1. [MEMORY.md](MEMORY.md) — the index the agent reads first, every session
2. [financial-profile.md](financial-profile.md) — cash flow, balance sheet, hard limits
3. [goals.md](goals.md) — goals with funding plans
4. [portfolio.md](portfolio.md) — holdings the agent keeps current
5. [thesis-exampleco.md](thesis-exampleco.md) — one filled thesis with a kill condition
6. [2026-06-15.md](2026-06-15.md) — a daily log from a day with an order

If your `memory/` folder looks structurally like this after onboarding, the system is working.
